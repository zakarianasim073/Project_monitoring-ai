## 2025-05-15 - Optimizing Batch Updates and Parallelizing Writes in Bill Creation
**Learning:** In scenarios involving auto-distribution of costs or amounts (like CLIENT_RA bills), replacing sequential document updates within a loop with a single `updateMany` call using `$inc` eliminates N+1 database round-trips. Additionally, since Mongoose generates document IDs locally, the initial `.save()` call can be parallelized with other independent updates (like linking the ID to a parent document) using `Promise.all` to significantly reduce total request latency.
**Action:** Always look for opportunities to replace fetch-and-save patterns in loops with atomic batch updates (`updateMany`, `bulkWrite`) and parallelize independent database writes using `Promise.all` when the new document ID is already available.
